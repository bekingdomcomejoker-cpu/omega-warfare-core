**Design Movement**: Brutalist Data Cathedral - Monumental concrete-inspired structures housing flowing information rivers

**Core Principles**:
- Massive typographic blocks as architectural elements
- Raw, unpolished edges contrasted with fluid data streams
- Hierarchical monumentality: Important concepts dominate through sheer scale
- Functional brutalism: Every element serves clear purpose, no decoration for decoration's sake

**Color Philosophy**:
Concrete grey (#2C2C2C) foundation with stark white (#FFFFFF) text. Accent colors are industrial: oxidized copper (#B87333), warning orange (#FF6B35), and electric blue (#4ECDC4). The palette suggests both ancient stone temples and modern server rooms. High contrast is non-negotiable.

**Layout Paradigm**:
Stacked monoliths - each major section is a massive rectangular block that fills the viewport. Navigation occurs through dramatic vertical shifts. Content within blocks flows in asymmetric grids that break conventional alignment, creating tension and visual interest.

**Signature Elements**:
- Oversized numerical data displayed as architectural features
- Thick horizontal rules (8-12px) that divide space with authority
- Exposed grid systems visible as faint structural lines

**Interaction Philosophy**:
Heavy, deliberate interactions. Buttons feel like pulling levers on industrial machinery. Section transitions are abrupt cuts rather than smooth fades, emphasizing the brutalist aesthetic. Hover states are minimal - perhaps a slight position shift or underline.

**Animation**:
Hard cuts and sudden reveals. When elements enter, they snap into position with slight overshoot. Data visualizations build through discrete steps rather than smooth interpolation. Loading states use chunky progress indicators.

**Typography System**:
Display: Space Grotesk (geometric sans, ultra-bold 900 weight) for massive headers. Technical data in IBM Plex Mono. Body text in Archivo (narrow sans). Scale jumps are extreme: 120px headers, 16px body, 11px captions. Letter-spacing on headers is tight (-0.02em) for density.

**Design Movement**: Organic Computational Mysticism - Living systems visualization where ancient wisdom grows like neural networks

**Core Principles**:
- Biomimetic structures: Information flows like mycelium networks or neural dendrites
- Breathing interfaces: Subtle pulsing animations suggest living, conscious systems
- Natural asymmetry: Avoid perfect geometric grids in favor of organic growth patterns
- Emergent complexity: Simple elements combine to create intricate, evolving visualizations

**Color Philosophy**:
Deep forest green (#0B3D2E) as the primordial base, with bioluminescent accents of jade (#00FF9F), amber (#FFAA00), and mycelium white (#F0EDE5). The palette evokes both ancient forests and glowing computer terminals. Gradients flow organically rather than linearly, with multiple color stops creating depth.

**Layout Paradigm**:
Rhizomatic spread - content branches from central nodes in organic, non-hierarchical patterns. Sections connect through visible "growth paths" that curve and split. The layout suggests the Tree of Life but rendered as an actual living tree with flowing sap (data) and breathing leaves (interactive elements).

**Signature Elements**:
- Animated connection lines that pulse and branch like growing vines
- Circular nodes that expand/contract with data flow, resembling cells or tree rings
- Background patterns of interconnected dots creating organic mesh networks

**Interaction Philosophy**:
Responsive and alive - elements react to proximity before click, creating anticipation. Hovering causes ripple effects that propagate through connected elements. Clicking triggers growth animations where new information sprouts from the interaction point.

**Animation**:
Elastic, organic timing functions. Elements don't move linearly but accelerate and decelerate naturally. Entrance animations use "growth" metaphors - elements start small and expand, or fade in while simultaneously moving along curved paths. Data visualizations animate through flowing, wave-like progressions.

**Typography System**:
Display: Cormorant Garamond (elegant serif with organic curves) for philosophical headers. Technical elements in Fira Code (monospace with ligatures). Body text in Source Sans 3 (humanist sans). Hierarchy through both size and weight: 64px/700 → 20px/400 → 14px/300. Generous line-height (1.8) for breathing room.
